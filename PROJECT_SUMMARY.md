# Project Summary: Academic Research Workflow Agent v1.0

## 1. 项目定位

Academic Research Workflow Agent 是一个面向学术论文阅读、文献整理、综述准备和写作检查的本地化研究工作流 Agent。项目基于 RAG 技术路线，将文档解析、文本切分、向量检索、大模型调用、结构化学术任务、前端交互和结果导出整合为一个可运行系统。

项目当前版本为 v1.0，已经完成阶段性收口。

## 5. 关键技术路径

项目主要技术路径包括：

1. 文档解析：读取 PDF、DOCX、TXT、MD；
2. 文本切分：将长文档切分为可检索 chunk；
3. 向量索引：使用 Chroma 存储文本片段；
4. RAG 检索：基于语义检索获取相关片段；
5. LLM 调用：使用真实 API 生成结构化回答；
6. 学术任务封装：精读、对比、综述、写作检查；
7. 前端交互：Streamlit 任务入口和下载按钮；
8. 结果导出：Markdown 与 Word；
9. 自动验收：evaluate_project.py。

---

## 6. 关键优化记录

### 6.1 前端上传与自动索引

模块十七实现了前端上传文档并自动重建索引，使系统从“手动复制文档”转为“前端上传即用”。

### 6.2 单篇论文精读检索优化

模块十八提升了英文论文精读能力，引入中英文关键词、结构性页码偏好和增强检索策略，减少“未找到明确依据”的情况。

### 6.3 表格型 chunk 检索增强

模块十九针对“数据与方法”“核心变量”等维度增加表格型 chunk 检索，改善变量表、指标表、模型公式和 DEA 指标体系的召回能力。

### 6.4 批量论文精读

模块二十实现了批量论文精读后端工具和 Streamlit 前端入口，支持一次选择多篇论文并生成 quick / full 精读结果。

### 6.5 导出结构优化

模块二十一优化了批量精读导出格式，使 Markdown 和 Word 更接近正式阅读报告结构。

## 8. 当前不足

当前版本仍存在以下限制：

1. 不支持扫描版 PDF OCR；
2. 不做真正的 PDF 表格结构化抽取；
3. 文献综述框架不等同于完整综述正文；
4. 写作检查不会自动验证参考文献真实性；
5. 当前没有任务历史记录；
6. 当前没有用户系统和在线部署；
7. Word 导出为基础格式优化，不是完整模板排版系统；
8. evaluate_project.py 主要验证功能可用性，不等同于完整单元测试体系。
